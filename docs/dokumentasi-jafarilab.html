1. Ringkasan Proyek

JafariLab adalah website portofolio kreatif dengan halaman publik dan panel administrasi terproteksi. Pengunjung dapat melihat profil, layanan, portofolio, statistik, serta informasi kontak. Administrator dapat mengelola proyek, layanan, dan hampir seluruh teks halaman tanpa mengubah source code.

KomponenFungsi
Website publikMenampilkan konten profil, hero, keunggulan, layanan, portofolio, tentang, statistik, kontak, dan footer.
Panel adminLogin, CRUD proyek, CRUD layanan, serta penyuntingan pengaturan website.
REST APIPenghubung browser dengan data MySQL dalam format JSON.
MySQLMenyimpan administrator, proyek, layanan, dan pengaturan key-value.
Upload lokalMenyimpan gambar portofolio di public/uploads/.

3. Arsitektur Sistem

Browser publik/admin→ HTTP/JSON →Express routes→ SQL →MySQL

Express melayani file statis dari folder public. JavaScript browser mengambil data melalui endpoint /api/*. Route publik menggunakan query baca, sedangkan route mutasi admin melewati middleware requireAuth. Gambar diunggah ke disk lokal dan path-nya disimpan dalam tabel projects.

Alur startup

  1. index.js memuat environment.
  2. Middleware JSON, form, session, dan static file dipasang.
  3. initDatabase() membuat database/tabel bila belum ada.
  4. Migrasi SQLite dijalankan bila relevan.
  5. Data awal dan akun admin dibuat bila tabel masih kosong.
  6. Express mendengarkan pada port dari PORT.

5. Persyaratan dan Instalasi

Persyaratan

  • Node.js dan npm.
  • MySQL Server aktif.
  • Akun MySQL dengan izin membuat database dan tabel.
  • Windows PowerShell, Terminal, atau shell setara.

Langkah instalasi

cd C:\Users\aeste\JafariLab
npm install

Salin atau buat .env, isi koneksi database dan kredensial awal. Pastikan MySQL aktif, lalu:

npm start
# sama dengan:
node index.js

Buka http://localhost:3000 bila PORT=3000.

9. Alur Autentikasi

  1. Admin mengirim username/password ke POST /api/auth/login.
  2. Server mencari username dengan prepared statement.
  3. bcryptjs membandingkan password dengan hash.
  4. Jika benar, adminId dan username disimpan pada session.
  5. Browser menerima cookie session dengan httpOnly, sameSite=lax, masa aktif 8 jam.
  6. Endpoint mutasi memanggil requireAuth.
  7. Logout menghancurkan session.

GET /admin menampilkan login atau dashboard berdasarkan session. GET /admin/dashboard mengalihkan pengguna tanpa session kembali ke login.

11. Panel Admin dan CMS

Panel admin diakses melalui /admin. Navigasi terdiri dari Ringkasan, Portofolio, Layanan, dan Konten Website.

  • Portofolio: tambah, edit, hapus, upload gambar, atur urutan.
  • Layanan: tambah, edit, hapus, atur urutan.
  • Konten website: identitas, topbar, hero, keunggulan, judul section, tentang, statistik, kontak, sosial, dan footer.

public/js/admin.js memuat data secara paralel, mengisi form berdasarkan atribut name, dan menyimpan melalui API. Penambahan field setting baru membutuhkan sinkronisasi di tiga tempat: field HTML admin, whitelist route settings, dan default seed/database.

12. Frontend Publik

public/index.html menyediakan struktur dan fallback content. Setelah halaman dimuat, public/js/app.js mengambil settings, proyek, serta layanan secara paralel. Konten kemudian dimasukkan ke DOM. Proyek dan layanan dirender berdasarkan jumlah record sehingga tidak dibatasi tiga item.

Fitur browser meliputi menu responsif, animasi IntersectionObserver, link proyek opsional, link email, dan link media sosial. CSS utama berada di public/css/style.css, dengan aturan admin terpisah di public/css/admin.css.

13. Pengelolaan Upload

  • Jenis file: JPEG, PNG, WebP, dan GIF.
  • Batas ukuran: 5 MB.
  • Nama file: timestamp + random bytes + ekstensi asli.
  • Lokasi: public/uploads/.
  • Path publik disimpan sebagai /uploads/nama-file.ext.
Catatan pemeliharaan: penghapusan proyek atau penggantian gambar belum menghapus file lama dari disk. Buat proses garbage collection berkala agar folder upload tidak terus membesar.

14. Panduan Coding

Prinsip umum

  • Gunakan ES Modules: import/export, bukan require.
  • Pisahkan routing, middleware, database, UI, dan aset berdasarkan tanggung jawab.
  • Semua query dengan input pengguna wajib memakai placeholder ? dan prepared statement.
  • Endpoint mutasi admin wajib memakai requireAuth.
  • Validasi input di server; validasi HTML hanya bantuan UX.
  • Jangan mengirim stack trace atau konfigurasi rahasia ke client.
  • Gunakan transaksi untuk perubahan banyak record yang harus atomik.
  • Jaga response API konsisten: success, data, dan/atau message.

Konvensi penamaan

AreaKonvensiContoh
JavaScriptcamelCase untuk variabel/fungsihydrateContent
Databasesnake_casesort_order
Setting keyssnake_case dan prefix sectionhero_description
CSSkebab-case, class semantikproject-visual
Route fileplural sesuai resourceservices.js

Pola route yang disarankan

router.post('/api/admin/resource', requireAuth, async (req, res, next) => {
  try {
    const value = String(req.body.value || '').trim();
    if (!value) {
      return res.status(400).json({ success: false, message: 'Value wajib diisi.' });
    }
    const [result] = await db.execute(
      'INSERT INTO resources (value) VALUES (?)',
      [value]
    );
    res.json({ success: true, id: result.insertId });
  } catch (error) {
    next(error);
  }
});

Aturan frontend

  • Gunakan textContent untuk teks dari API. Jika harus memakai innerHTML, lakukan escaping.
  • Jangan membangun URL sensitif langsung dari input tanpa validasi protocol.
  • Selalu periksa response.ok dan tangani 401 dengan redirect login.
  • Jaga progressive enhancement: HTML fallback tetap dapat dibaca ketika API gagal.
  • Pastikan elemen baru responsif pada breakpoint yang ada.

Format dan kualitas

Source saat ini cukup ringkas. Untuk pengembangan tim, gunakan ESLint + Prettier, pecah fungsi panjang, dan tambahkan JSDoc hanya pada fungsi dengan kontrak yang tidak jelas. Hindari komentar yang sekadar mengulang kode.

15. Pola Penambahan Fitur

Menambah field CMS

  1. Tambahkan input dengan name="key_baru" pada dashboard admin.
  2. Tambahkan key ke whitelist di routes/settings.js.
  3. Tambahkan nilai default di config/database.js.
  4. Hubungkan key ke elemen publik di public/js/app.js.
  5. Perbarui database/jafarilab.sql dan dokumentasi.
  6. Uji load, simpan, reload browser, dan restart server.

Menambah resource CRUD

  1. Buat/migrasikan tabel.
  2. Buat file route dengan GET publik dan mutasi admin.
  3. Pasang route di index.js.
  4. Tambahkan UI admin dan state/rendering.
  5. Tambahkan rendering publik jika diperlukan.
  6. Uji happy path, input salah, 401, 404, dan error database.

16. Validasi dan Pengujian

Project belum mempunyai automated test; script npm test masih placeholder. Minimal verification sebelum rilis:

  • node --check untuk seluruh file JavaScript server dan browser.
  • Server berhasil startup dan terhubung MySQL.
  • GET /, settings, portfolio, dan services menghasilkan 200.
  • Login salah menghasilkan 401; login benar membuat session.
  • CRUD proyek dan layanan berfungsi, termasuk upload valid/terlalu besar.
  • Pengaturan CMS tersimpan dan tampil setelah reload.
  • Logout mengakhiri akses endpoint admin.
  • Uji tampilan desktop dan mobile.

Roadmap test

Gunakan Node test runner atau Vitest untuk unit test, Supertest untuk integration API, dan Playwright untuk alur login/CMS. Gunakan database test terpisah dan bersihkan fixture setelah setiap suite.

17. Keamanan

Yang sudah diterapkan

  • Password di-hash dengan bcrypt.
  • Query input menggunakan parameter.
  • Cookie session httpOnly dan sameSite=lax.
  • Mutasi data dilindungi middleware autentikasi.
  • Upload dibatasi tipe MIME dan ukuran.
  • Setting yang dapat diubah dibatasi whitelist.

Wajib sebelum produksi

  • Ganti MemoryStore session dengan MySQL/Redis session store.
  • Aktifkan HTTPS dan cookie.secure=true; set app.set('trust proxy', 1) di belakang proxy.
  • Gunakan SESSION_SECRET, password admin, dan DB password yang kuat.
  • Pasang Helmet, rate limiting pada login, CSRF protection, dan audit log.
  • Validasi URL agar hanya menerima http:/https:.
  • Scan konten upload berdasarkan isi file, bukan hanya MIME client.
  • Gunakan user MySQL dengan least privilege, bukan root.
  • Jangan tampilkan panel admin tanpa kontrol akses jaringan bila tidak diperlukan.

18. Deployment dan ngrok

Preview melalui ngrok

  1. Jalankan aplikasi dan pastikan http://localhost:3000 aktif.
  2. Konfigurasikan token ngrok sekali: ngrok config add-authtoken TOKEN.
  3. Jalankan ngrok http 3000.
  4. Gunakan URL HTTPS yang ditampilkan.
URL ngrok membuka aplikasi lokal ke internet. Gunakan password admin kuat, jangan membagikan token, dan hentikan tunnel setelah selesai.

Deployment produksi

Gunakan process manager seperti PM2 atau service manager OS, reverse proxy Nginx/Apache, HTTPS, database terkelola, session store persisten, dan penyimpanan upload persisten/object storage. Pastikan folder upload tidak hilang saat deploy container baru.

Checklist environment produksi

  • Environment rahasia tersimpan di secret manager.
  • HTTPS dan domain aktif.
  • Backup database dan upload terjadwal.
  • Log error dan uptime monitoring aktif.
  • Cookie production aman.
  • Firewall hanya membuka port yang dibutuhkan.

19. Backup dan Pemeliharaan

Backup MySQL

mysqldump -u USER -p --single-transaction jafarilab > jafarilab-backup.sql

Restore

mysql -u USER -p jafarilab < jafarilab-backup.sql

Backup database harus dipasangkan dengan salinan folder public/uploads. Uji restore berkala; backup yang belum pernah diuji belum dapat dianggap aman.

Rutinitas

  • Perbarui dependency secara terkontrol dan audit kerentanan.
  • Rotasi password dan secret.
  • Bersihkan file upload yatim.
  • Pantau ukuran database, disk, log, error 5xx, dan login gagal.
  • Jalankan smoke test setelah setiap perubahan.

21. Keterbatasan dan Roadmap

  1. Session produksi: ganti MemoryStore.
  2. Automated tests: bangun integration dan E2E suite.
  3. Upload lifecycle: hapus file lama/yatim dan gunakan storage persisten.
  4. Security hardening: CSRF, Helmet, rate limit, secure cookie, audit log.
  5. Role management: saat ini hanya satu tipe administrator.
  6. Schema migration: gunakan migration tool, bukan hanya CREATE TABLE.
  7. Observability: structured logging, health endpoint, metrics, alerting.
  8. SEO CMS: title, meta description, OG image, sitemap.
  9. Encoding cleanup: normalisasi string lama yang menunjukkan mojibake.

22. Checklist Serah Terima

  • Source code dan lockfile tersedia.
  • .env.example tersedia tanpa rahasia.
  • Akun admin produksi diserahkan melalui kanal aman.

  • Database dan upload sudah dibackup.
  • Domain, HTTPS, reverse proxy, dan process manager terdokumentasi.
  • Login, CRUD, CMS, email, sosial, dan responsive view diuji.
  • Prosedur restore dan rollback diuji.
  • Pemilik sistem memahami batasan session/upload saat ini.
